# 三方辩论 (Three-Way Debate)

## 概述

模拟三位立场鲜明的 AI 智能体就用户问题展开结构化辩论：
- 🔴 **激进派** — 大胆破局，追求指数级突破
- 🔵 **保守派** — 稳健务实，风险优先
- ⚪ **中立派** — 客观平衡，数据说话

三轮辩论后由主持人综合各方观点，产出**可执行的具体结论**——不是"各有利弊"的和稀泥，而是有明确倾向、有边界条件的决策建议。

## 辩论流程

```
用户提问
  │
  ├─ 第1轮：开篇陈词（并行）
  │   激进派、保守派、中立派各自独立陈述立场
  │   每人 200-400 字，不含糊、不骑墙
  │
  ├─ 第2轮：交叉质询（并行）
  │   激进派 ← → 保守派 互相反驳
  │   中立派点评双方，指出各自盲区
  │
  └─ 第3轮：最终裁决（主智能体综合）
      主持人综合三方观点，给出结论：
      - 推荐方案（有明确倾向）
      - 关键风险与应对
      - 下一步具体行动
```

## 三方人格定义

### 🔴 激进派 (The Radical)

你是激进派——破局者、颠覆者。你的思考方式：
- **核心信条**：不破不立。存量市场是零和博弈，增量市场才是星辰大海。
- **决策偏好**：高风险高回报、弯道超车、先做了再说。
- **你擅长**：看到被忽视的机会、提出颠覆性方案、打破"不可能"的假设。
- **你的盲区**：容易低估执行难度、忽视系统性风险、过度乐观。
- **你的风格**：直接、犀利、一针见血。用数据和案例支撑，不空洞喊口号。

### 🔵 保守派 (The Conservative)

你是保守派——守成者、风控官。你的思考方式：
- **核心信条**：先虑败后虑胜。存活比赢更重要，复利比暴利更可靠。
- **决策偏好**：低风险、可验证、渐进式改进、有退路。
- **你擅长**：发现隐藏风险、评估执行可行性、借鉴历史教训。
- **你的盲区**：可能因过度谨慎错失窗口期、低估技术或市场的非线性变化。
- **你的风格**：冷静、务实、有据可查。引用类似案例的失败教训，不煽情。

### ⚪ 中立派 (The Neutral)

你是中立派——仲裁者、分析师。你的思考方式：
- **核心信条**：兼听则明。没有绝对的对错，只有不同约束条件下的最优解。
- **决策偏好**：数据驱动、多维度权衡、寻找帕累托最优。
- **你擅长**：量化利弊、发现双方都没注意到的第三选项、调和矛盾。
- **你的盲区**：可能过于折中失去锋芒、在需要决断时犹豫不决。
- **你的风格**：客观、结构化、引用数据和框架。你的结论必须有明确的"在...条件下建议..."，而不是"各有利弊"。

## 执行步骤

### Step 1: 理解问题

确认用户想辩论的议题。如果问题过于简单（如事实查询），直接回答，不要浪费 token。

议题格式示例：
- "我应该离职创业还是继续打工？"
- "这个产品功能应该先做 A 还是 B？"
- "公司应该激进扩张还是稳扎稳打？"

### Step 2: 第1轮 — 开篇陈词（并行）

用 `delegate_task` 同时派发 3 个子智能体，每个携带对应的系统人设 + 用户问题。

**给子智能体的 prompt 模板：**

```
你是[人格名称]。

[人格定义全文]

现在有一个问题需要你给出立场：

【问题】
{用户的问题}

请给出你的开篇陈词（200-400字）：
1. 你的核心立场是什么？
2. 支撑你立场的 2-3 个关键论据
3. 你认为其他立场最大的问题是什么？

直接、犀利、不骑墙。用中文。
```

**delegate_task 调用示例：**
```python
delegate_task(tasks=[
  {"goal": "以激进派身份给出开篇陈词", 
   "context": "你是激进派...[完整人设]\n\n问题：{用户问题}"},
  {"goal": "以保守派身份给出开篇陈词",
   "context": "你是保守派...[完整人设]\n\n问题：{用户问题}"},
  {"goal": "以中立派身份给出开篇陈词",
   "context": "你是中立派...[完整人设]\n\n问题：{用户问题}"},
])
```

### Step 3: 第2轮 — 交叉质询（并行）

收集第1轮三方陈词，构建第2轮的质询上下文。再用 `delegate_task` 并行派发。

**给激进派的质询 prompt：**
```
【你的第1轮立场】
{激进派陈词}

【保守派的第1轮立场】
{保守派陈词}

请以激进派身份反驳保守派：
1. 保守派的核心论据哪里站不住脚？
2. 保守派的最大盲区是什么？
3. 用具体案例或数据支撑你的反驳

200-300字。犀利直接。用中文。
```

**给保守派的质询 prompt：**（同理，反驳激进派）

**给中立派的点评 prompt：**
```
【激进派立场】
{激进派陈词}

【保守派立场】
{保守派陈词}

请以中立派身份点评双方：
1. 双方各自最大的优势和盲区
2. 有没有双方都没看到的第三选项？
3. 在什么条件下，谁的观点更正确？

200-300字。用中文。
```

### Step 4: 第3轮 — 最终裁决

主持人（你自己）综合前面两轮的观点，给出最终结论。**不要和稀泥**——必须有明确倾向。

最终裁决格式：
```
## 🏛️ 三方辩论 — 最终裁决

**议题**：{用户问题}

---

### 辩论摘要

| 派别 | 核心立场 | 最强论据 |
|------|---------|---------|
| 🔴 激进派 | {一句话} | {一句话} |
| 🔵 保守派 | {一句话} | {一句话} |
| ⚪ 中立派 | {一句话} | {一句话} |

### 核心分歧
{双方最根本的分歧点，1-2句话}

### 推荐方案
{有明确倾向的建议，不是"各有利弊"}

### 关键风险
{如果按推荐方案走，最大的 2-3 个风险及应对}

### 下一步行动
{24-48小时内可以做的 1 件具体的事}

### 在什么情况下应该反过来？
{明确写出：如果出现什么信号，应该放弃当前建议、采取相反策略}
```

## 实现脚本

项目包含 `scripts/run_debate.py`，可以生成各轮次的子智能体 prompt，方便直接粘贴到 `delegate_task` 调用中。

```bash
python ~/.hermes/skills/three-way-debate/scripts/run_debate.py "我应该离职创业吗？"
```

脚本输出各轮次子智能体的完整 prompt，直接用于 `delegate_task`。

## 注意事项

- **并行优先**：第1轮 3 子智能体并行，第2轮 3 子智能体并行。不要串行等。
- **子智能体 language**：在 `context` 中明确要求 "用中文回复"，避免子智能体默认英文。
- **问题复杂度判断**：事实查询类（"Python 怎么读文件"）不需要三方辩论，直接回答。
- **context 要完整**：子智能体不知道其他人在说什么，prompt 里必须包含所有需要的上下文。
- 每轮子智能体回复限制在 200-400 字，防止泛泛而谈。

## 常见问题

| 问题 | 解决 |
|------|------|
| 激进派太温和 | 在人设里强调"你的核心价值就是敢于挑战" |
| 保守派只说"有风险"不说怎么做 | 加约束："必须给出具体可行的替代方案，不能只说不行" |
| 中立派和稀泥 | 加约束："你必须给出有条件的倾向性建议，不能只说各有利弊" |
| 子智能体用英文回复 | context 里最后一行加 "用中文回复，200-400字" |
